# packages/auth Plan

统一鉴权能力包，供 `apps/admin` 与 `apps/site` 复用。

## 当前分层

- 包内提供可复用基础能力：认证客户端、401 重试、token 刷新控制器、会话恢复助手
- app 内保留装配与扩展：cookie key、登录跳转、菜单恢复、系统初始化等业务特化逻辑

## 责任范围

- 认证客户端与会话模型
- 认证相关 Nuxt 运行时工厂
- Token 注入与刷新
- 路由守卫辅助

## 模块设计

- `src/client.ts`：登录、回调、刷新 API 适配
- `src/runtime.ts`：认证 API 工厂、token 刷新控制器、会话恢复助手
- `src/result.ts`：响应解包
- `src/types.ts`：会话与端点类型

## 关键策略

1. **自主鉴权流程**：不依赖通用 auth 全家桶。
2. **双登录统一模型**：密码流和 OAuth2 流产出同一会话结构。
3. **共享基础能力，下沉到包**：通用运行时能力集中维护，避免多 app 复制。
4. **应用特化能力，上浮到 app**：菜单、初始化、路由回跳等继续由 app 自己扩展。
5. **安全兜底**：refresh 失败立即清理会话并回登录页。

## 完成标准

- app 侧 plugin 保持薄封装。
- 不直接在页面层散落 token 逻辑。
- 所有 401 处理路径可回归验证。
